# Support Ticket Classification System

## Overview

This project implements an end-to-end **Machine Learning pipeline** that automatically classifies customer support tickets into meaningful categories (e.g. Billing, Technical Issue, Account Management) and assigns a priority level (High / Medium / Low). The system is designed to reduce manual triage effort, accelerate response times, and ensure high-priority issues are never buried in a queue.

The pipeline covers every stage of a real-world NLP workflow: raw text cleaning, tokenisation, lemmatisation, TF-IDF feature extraction, multi-class classification with Logistic Regression, model serialisation, and a live prediction API — all in pure Python / scikit-learn.

## Business Value

- **Faster response** — automated triage cuts average ticket-handling time significantly
- **Automated routing** — tickets are instantly directed to the right team (Billing, Tech Support, etc.)
- **Priority handling** — urgent tickets are flagged and escalated before agents even see them
- **Scalability** — the pipeline can process thousands of tickets per minute without extra headcount
- **Consistency** — removes human bias; every ticket is classified by the same rules every time
- **Cost reduction** — fewer Tier-1 agents needed; focus human effort on complex, high-value issues

## Dataset

| Field | Detail |
|---|---|
| Source | [Kaggle — Customer Support Ticket Dataset](https://www.kaggle.com/datasets/suraj520/customer-support-ticket-dataset) |
| File | `data/customer_support_tickets.csv` |
| Key columns | `Ticket Description` (text), `Ticket Type` (category), `Ticket Priority` (priority) |
| Size | ~8,000 labelled support tickets |

Place the downloaded CSV at `data/customer_support_tickets.csv` before running.

## How It Works

```
Raw ticket text
     │
     ▼
[TextPreprocessor]       lowercase → remove URLs/emails → strip punctuation
     │                   → tokenise → remove stopwords → lemmatise
     ▼
[FeatureExtractor]       TF-IDF with unigrams + bigrams, top-5000 features
     │
     ▼
[Logistic Regression]    Category Classifier  ┐
[Logistic Regression]    Priority Classifier  ┘  trained simultaneously
     │
     ▼
  Prediction + Confidence + Routing Recommendation
```

## Models Used

| Model | Role |
|---|---|
| **Logistic Regression** | Primary model — fast, explainable, strong on text |
| **Multinomial Naive Bayes** | Secondary baseline — great on sparse BoW features |
| **Random Forest** | Ensemble option — available via `train_model(model_type='rf')` |


## How to Run


### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Place the dataset CSV
Download the dataset from Kaggle and place it at:
```
data/customer_support_tickets.csv
```

### 3. Train the Model
```bash
python main.py
```
This will preprocess the data, train the classifiers, and save the models to the `models/` directory.

### 4. (Optional) Run EDA
```bash
python notebooks/01_explore_data.py
```

### 5. (Optional) Post-training analysis
```bash
python notebooks/02_model_analysis.py
```

### 6. Run the Streamlit Web App
```bash
streamlit run app.py
```
This launches an interactive web dashboard for ticket classification. Paste or type a support ticket, click 'Classify Ticket', and view the predicted category, priority, and routing recommendation.
